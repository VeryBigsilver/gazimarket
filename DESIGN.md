# 1. 요구사항 도출

본 프로젝트는 사용자가 안전하게 중고 물품을 거래할 수 있는 웹 기반 중고거래 플랫폼을 개발하는 것을 목표로 한다. 사용자는 회원가입 후 상품을 등록하고 검색할 수 있으며, 다른 사용자와 채팅 및 송금을 통해 거래를 진행할 수 있다. 또한 관리자는 신고 기능을 통해 악성 사용자와 부적절한 상품을 관리하여 안전한 거래 환경을 제공한다.
## 기능 요구사항
### 1. 회원 관리
* 사용자는 회원가입 및 로그인을 할 수 있어야 한다.
* 사용자는 자신의 회원 정보를 조회 및 수정할 수 있어야 한다.
### 2. 상품 관리
* 사용자는 상품을 등록할 수 있어야 한다.
* 등록한 상품을 수정하거나 삭제할 수 있어야 한다.
* 모든 사용자는 등록된 상품을 검색하고 조회할 수 있어야 한다.
### 3. 거래 기능
* 사용자 간 송금을 수행할 수 있어야 한다.
* 거래 완료 여부를 확인할 수 있어야 한다.
### 4. 사용자 소통
* 사용자 간 1:1 채팅을 할 수 있어야 한다.
* 전체 사용자가 참여할 수 있는 공용 채팅 기능을 제공해야 한다.
### 5. 신고 기능
* 사용자는 악성 사용자 또는 부적절한 상품을 신고할 수 있어야 한다.
* 신고 시 신고 사유를 입력할 수 있어야 한다.
### 6. 관리자 기능
* 관리자는 회원 정보를 관리할 수 있어야 한다.
* 관리자는 등록된 상품을 관리할 수 있어야 한다.
* 관리자는 신고 내역을 확인할 수 있어야 한다.
* 관리자는 신고가 누적된 상품을 삭제할 수 있어야 한다.
* 관리자는 신고가 누적된 사용자를 휴면 또는 이용 제한 상태로 변경할 수 있어야 한다.

## 비기능 요구사항
* 사용자 개인정보는 안전하게 보호되어야 한다.
* 비밀번호는 암호화하여 저장해야 한다.
* 웹 서비스는 안정적으로 동작해야 한다.
* 사용자 입력에 대한 검증을 수행하여 보안 취약점을 방지해야 한다.
* 다수의 사용자가 동시에 서비스를 이용할 수 있어야 한다.

# 2. 시스템 설계
## 2.1 회원 관리
플랫폼 이용을 위해 회원가입과 로그인 기능을 제공한다.
* 로그인 페이지
  * 아이디와 비밀번호를 입력하여 로그인
  * 회원가입 페이지로 이동 가능
* 회원가입 페이지
  * 아이디, 비밀번호, 이름, 이메일을 입력하여 회원가입
  * 아이디 중복 여부 확인
* 마이페이지
  * 사용자 프로필 조회
  * 소개글 수정
  * 비밀번호 변경
  * 내가 등록한 상품 조회
  * 거래 내역 조회
회원 정보는 데이터베이스에 저장하여 관리한다.
## 2.2 상품 관리
사용자는 상품을 등록하고 조회할 수 있다.
* 메인 페이지
  * 등록된 상품 목록 출력
  * 상품 검색 기능 제공
* 상품 등록 페이지
  * 상품명
  * 가격
  * 상품 설명
  * 상품 사진 업로드
* 상품 상세 페이지
  * 상품 사진
  * 상품명
  * 가격
  * 상품 설명
  * 판매자 정보
  * 채팅 버튼
  * 신고 버튼
* 상품 관리 페이지
  * 내가 등록한 상품 조회
  * 상품 수정
  * 상품 삭제
상품 정보는 데이터베이스에 저장하여 관리한다.
## 2.3 거래 기능
사용자는 플랫폼 내에서 다른 사용자와 거래를 진행할 수 있다.
* 사용자 간 가상 포인트 송금
* 거래 완료 처리
* 거래 내역 조회
## 2.4 사용자 소통
거래를 원활하게 진행하기 위해 채팅 기능을 제공한다.
* 전체 채팅
  * 모든 사용자가 참여 가능한 공개 채팅
* 1:1 채팅
  * 판매자와 구매자가 개별적으로 대화 가능
채팅 내역은 데이터베이스에 저장한다.
## 2.5 신고 및 관리자 기능
안전한 거래 환경을 위해 신고 시스템과 관리자 기능을 제공한다.
### 신고 기능
* 상품 신고
* 사용자 신고
* 신고 사유 입력
* 신고 내역 저장
### 관리자 기능
* 회원 목록 조회
* 상품 목록 조회
* 신고 내역 조회
* 신고된 상품 삭제
* 신고 누적 사용자 휴면 처리
* 신고 누적 상품 차단
## 2.6 데이터 관리
플랫폼에서 사용하는 주요 데이터는 데이터베이스를 통해 관리한다.
* 회원 정보
* 상품 정보
* 채팅 내역
* 거래 내역
* 신고 내역
